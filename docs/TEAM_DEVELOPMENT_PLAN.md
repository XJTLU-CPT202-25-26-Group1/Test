# Team Development Plan

## Suggested 8-Person Split

1. Auth and Security
2. User and Profile
3. Specialist and Category
4. Availability
5. Customer Booking Flow
6. Admin Booking Review
7. Feedback and Billing
8. Front-end Integration and Testing

## Ownership Rule

Each person should mainly own:

- one controller area
- one service area
- one or more DTOs
- related templates/static files

## Shared Review Rule

Changes to these files require team review:

- `User`
- `Role`
- `Booking`
- `AvailabilitySlot`
- `ExpertiseCategory`
- `SecurityConfig`
